# Amazon 利润全面分析

## 概述

利润全面分析技能对亚马逊产品的盈利能力进行逐层拆解，帮助卖家了解每笔销售中真实发生的所有成本项，计算真实的净利润率（而非简单的毛利润率），并识别隐藏的费用陷阱和优化机会。该技能支持单个ASIN级别的精细利润计算，也支持产品组合层面的整体盈利能力评估，涵盖美国、英国、德国、加拿大、日本、澳大利亚等所有亚马逊站点。

亚马逊的费用结构极其复杂，许多卖家仅关注佣金和FBA配送费这两项"显性成本"，却忽略了长期仓储费、移除订单费、退款损失、优惠券费用、广告成本分摊等"隐性成本"。这些被忽略的成本往往足以将一个看似盈利的产品拖入亏损境地。该技能的核心价值在于构建完整的"收入瀑布"模型，确保卖家对产品的真实盈利水平有清醒的认知。

## 核心原理

**利润瀑布（Revenue Waterfall）模型**：利润瀑布是一种从销售收入出发，逐层扣除各项成本，最终得到净利润的可视化分析模型。完整的亚马逊利润瀑布应包含以下层次：销售收入 -> 商品成本（COGS）-> 亚马逊销售佣金（通常为8%-15%）-> FBA配送费（基于商品尺寸重量分级计算）-> 广告费用分摊（ACoS/TACoS计算）-> 退货成本（退货率乘以单笔损失金额）-> 长期仓储费分摊 -> 移除/销毁费用分摊 -> 优惠券折扣成本 -> 其他杂费 -> 净利润。每一层都以金额和占销售收入百分比的形式展示，使卖家能够直观看到"钱都去哪了"。

**ACoS与TACoS的区别与影响**：ACoS（广告销售成本比）= 广告支出 / 广告带来的销售额，反映的是广告的直接效率。TACoS（总广告销售成本比）= 广告支出 / 总销售额（包括自然销售和广告销售），反映的是广告对整体业务的影响程度。一个ACoS为20%看似健康的广告活动，如果其带来的销售额仅占总销售额的10%，那么TACoS仅为2%，广告支出占比很低；但如果广告带来的销售额占总销售额的80%，那么TACoS就是20%，广告几乎吃掉了所有利润。该技能通过同时计算ACoS和TACoS，帮助卖家准确评估广告支出的真实影响。

**隐性费用识别机制**：亚马逊的费用体系中存在大量容易被忽视的隐性费用。长期仓储费（库存存放超过365天按立方英尺收取的费用，以及旺季附加费）、移除订单费（将库存从亚马逊仓库退回或销毁的费用）、退款管理费（亚马逊不退还已收取的佣金部分）、优惠券赎回费（每笔优惠券使用收取的固定费用）、低量库存费和库存绩效指标扣分费用等。这些费用单独来看金额不大，但累积起来可能显著侵蚀利润。该技能通过系统性检查清单确保所有隐性费用都被纳入计算。

**盈亏平衡分析模型**：盈亏平衡分析帮助卖家了解产品的"安全边界"——最低可接受的售价、最高可承受的广告ACoS、最高可容忍的退货率。当售价低于盈亏平衡点时，无论销量多高都无法盈利。该技能通过计算各维度的盈亏平衡阈值，帮助卖家设定定价底线和运营警戒线。

## 工作流程

1. **产品数据收集**：获取产品的基础信息，包括售价、商品成本、品类、尺寸重量（用于FBA费用计算）、月销量、广告支出和退货率等。
2. **补充信息确认**：通过多选问题确认其他可能影响利润的因素，如是否有优惠券、是否使用多渠道配送、库存周转速度等。
3. **完整利润瀑布构建**：基于收集到的所有数据，逐层计算每一项成本，构建完整的利润瀑布模型，计算出真实的净利润率。
4. **盈亏平衡与场景模拟**：计算各维度的盈亏平衡点，并进行场景模拟（如售价变动、费用上涨、广告预算调整对利润的影响）。
5. **优化建议输出**：按照对利润的影响金额排序，输出前三大优化机会，并提供具体的优化方案和预期效果。所有基于估算的数据标注警告。

## 数据来源

本技能的费用计算核心基于内置的 FBA 费用计算逻辑，该逻辑参考了 amazon-fba-fee-calculator 的费率表结构，涵盖销售佣金（按品类百分比计算）、FBA 配送费（基于商品尺寸和重量分级）、月度仓储费、长期仓储费等主要费用项。然而，这些内置费率为品类级别的估算值，并非亚马逊官方实时费率。亚马逊的 FBA 费用标准可能因站点、季节和政策调整而变化，内置数据可能滞后于实际费率，在计算结果中会标注估算警告。

本技能的核心财务数据完全由用户提供，包括产品售价、商品采购成本（COGS）、头程物流费用、广告支出金额或 ACoS 百分比、历史退货率、库存周转速度等。这些数据是利润计算的基础输入，计算结果的准确性直接取决于用户提供数据的真实性和完整性。技能本身不会从外部获取用户的实际销售数据或费用明细。

为获得精确的利润分析结果，建议用户从 Seller Central 导出实际的费用明细（如支付报告、FBA 费用明细报告和广告花费报告），将这些实际数据作为输入而非依赖估算。如果用户仅能提供部分数据，本技能将使用品类平均值作为缺失参数的替代，并在输出结果中明确标注哪些数据为估算值。用户还需要提供产品包装后的精确尺寸和重量，因为 FBA 费用的尺寸重量分级对最终费用计算影响显著。

## 局限性

- FBA配送费的计算依赖于商品的实际尺寸和重量分级（标准小件、标准大件、大件等），如果用户提供的数据不够精确（如未包含包装后的尺寸），计算结果可能存在偏差。
- 广告效果的 Attribution 窗口（归因窗口）会影响ACoS计算的准确性。亚马逊默认的14天归因窗口意味着某些广告点击带来的销售可能未被计入广告销售额。
- 退货成本的计算基于历史退货率的估算，实际退货成本还受到退货原因分布（如退货商品是否可二次销售）的影响，不同退货原因导致的损失差异很大。
- 跨国运营时，汇率波动、各国不同的税率政策和FBA费用标准会增加利润计算的复杂性。
- 季节性因素对FBA仓储费有显著影响（旺季10-12月仓储费率更高），需要卖家提供库存计划和周转预期才能准确预估。
- 某些隐性费用（如品牌注册费、软件工具订阅费等运营层面的固定成本）是否分摊到单个ASIN上，需要根据卖家的会计核算方式来确定。
